# import-compose

Convert a Docker Compose file into a bench configuration file.

## Usage

```bash
# Print to stdout
bench import-compose --file docker-compose.yml

# Write to file
bench import-compose --file docker-compose.yml --output bench.yml
```

## Flags

| Flag | Description |
|---|---|
| `--file PATH` | Path to docker-compose.yml (required) |
| `--output PATH` | Output file path (default: stdout) |

## Field mapping

| Compose field | bench.yml field | Notes |
|---|---|---|
| `services.<name>.image` | `container.image` | Required — services without image are skipped |
| `services.<name>.ports` | `container.ports` | Converted to `host:container` strings |
| `services.<name>.volumes` | `container.volumes` | Bind mounts only, `host:container[:ro]` format |
| `services.<name>.command` | `container.command` | Array form |
| `services.<name>.network_mode` | `container.network` | |
| `services.<name>.environment` | `env` | Map form |
| `services.<name>.env_file` | `env_file` | First entry only |
| `services.<name>.depends_on` | `depends_on` | Service names |
| `services.<name>.restart` | `restart.policy` | `no`/empty -> `never`, `on-failure` -> `on-failure`, `always`/`unless-stopped` -> `always` |
| `services.<name>.stop_signal` | `stop_signal` | |
| `services.<name>.stop_grace_period` | `shutdown_timeout` | |
| `services.<name>.labels` | `labels` | |
| `services.<name>.healthcheck` | `readiness` | Best-effort, see below |

## Healthcheck conversion

Compose healthchecks are converted to bench readiness checks on a best-effort basis:

- **HTTP**: If the healthcheck command contains `curl` or `wget` with an HTTP URL, it maps to `readiness.kind: http` with the extracted URL.
- **TCP**: If the command uses `nc` or `/dev/tcp/`, it maps to `readiness.kind: tcp` with the extracted address.
- **Other**: Commands that don't match these patterns are skipped with a warning. You'll need to configure readiness manually.

Healthcheck `timeout` maps to `readiness.timeout` and `start_period` maps to `readiness.initial_delay`.

## Skipped and unsupported fields

The following Compose features have no bench equivalent and produce warnings:

- **`build`**: Services with `build` but no `image` are skipped entirely. Services with both `build` and `image` use the image and ignore the build config.
- **`deploy`**: Ignored (bench doesn't manage replicas or resource constraints).
- **`configs` / `secrets`**: Ignored.
- **Multiple `env_file` entries**: Only the first is used.
- **Named volumes**: Only bind-mount volumes are converted.

## Example

Given this `docker-compose.yml`:

```yaml
services:
  postgres:
    image: postgres:16
    ports:
      - "5432:5432"
    environment:
      POSTGRES_DB: myapp
    restart: unless-stopped

  app:
    image: myapp:latest
    ports:
      - "8080:8080"
    depends_on:
      - postgres
    healthcheck:
      test: ["CMD-SHELL", "curl -f http://localhost:8080/health"]
      timeout: 5s
      start_period: 10s
```

Running `bench import-compose --file docker-compose.yml` produces:

```yaml
# Generated by: bench import-compose --file docker-compose.yml

version: 1
services:
  postgres:
    container:
      image: postgres:16
      ports:
        - 5432:5432
    env:
      POSTGRES_DB: myapp
    restart:
      policy: always
  app:
    container:
      image: myapp:latest
      ports:
        - 8080:8080
    depends_on:
      - postgres
    readiness:
      kind: http
      url: http://localhost:8080/health
      timeout: 5s
      initial_delay: 10s
```
